# Outputs Directory

**Created:** 2025-10-27
**Last Updated:** 2025-10-28
**Status:** Active - Mandatory for all agents
**Category:** Project Guidelines

---

## Purpose

This directory is the **MANDATORY central output location** for ALL content generated by BMAD agents. No agent shall create outputs in agent folders, command folders, or any other location.

---

## Mandatory Structure

### Directory Organization

```
outputs/
├── {MM-DD-YYYY}/                    # Date folder (TODAY'S DATE)
│   └── {session-name}/              # Unique session folder
│       ├── images/                  # Generated images (if applicable)
│       ├── videos/                  # Generated videos (if applicable)
│       ├── posts/                   # Generated text posts (if applicable)
│       ├── {content-name}.md        # Main output document
│       └── metadata.json            # Session metadata
```

### Real Example

```
outputs/
└── 10-28-2025/                      # Today's date
    ├── xai-twitter-thread/          # Session 1
    │   ├── images/
    │   │   └── xai-logo.png
    │   └── xai-twitter-thread-2025-10-28.md
    └── linkedin-carousel-ai-tools/  # Session 2
        ├── images/
        │   ├── slide1.png
        │   ├── slide2.png
        │   └── slide3.png
        └── linkedin-carousel-2025-10-28.md
```

## Naming Conventions

### Date Folders
- **Format:** MM-DD-YYYY (e.g., "10-28-2025")
- **Rule:** ONE folder per day
- **Creation:** Auto-created by agents if doesn't exist

### Session Folders
- **Format:** lowercase-kebab-case
- **Pattern:** {platform}-{content-type}-{topic}
- **Examples:**
  - `twitter-thread-xai-launch`
  - `linkedin-carousel-ai-agents`
  - `youtube-short-productivity-tips`
  - `instagram-reel-daily-routine`
- **Rule:** Unique name per content generation session
- **Tip:** Be descriptive enough to understand content at a glance

### Content Files
- **Format:** lowercase-kebab-case with date suffix
- **Pattern:** {content-type}-{topic}-{YYYY-MM-DD}.md
- **Examples:**
  - `twitter-thread-2025-10-28.md`
  - `linkedin-post-2025-10-28.md`
  - `youtube-script-2025-10-28.md`

## Agent Responsibilities

### MANDATORY Rules for All Agents

#### 1. Output Location
```yaml
MUST:
  - Create outputs in {project-root}/outputs/{MM-DD-YYYY}/{session-name}/
  - Generate today's date folder if it doesn't exist
  - Create unique session folder for each content generation

NEVER:
  - Create outputs in agent folders (bmad/agents/*/outputs/)
  - Create outputs in command folders (.claude/commands/*)
  - Create outputs in sidecar folders (bmad/agents/*/sidecar/outputs/)
  - Create outputs anywhere except {project-root}/outputs/
```

#### 2. Folder Creation Workflow
```bash
Step 1: Get today's date
  DATE=$(date +"%m-%d-%Y")  # e.g., "10-28-2025"

Step 2: Create date folder if needed
  mkdir -p "{project-root}/outputs/$DATE"

Step 3: Create unique session folder
  SESSION="linkedin-carousel-ai-agents"  # descriptive name
  mkdir -p "{project-root}/outputs/$DATE/$SESSION"

Step 4: Create content subfolders as needed
  mkdir -p "{project-root}/outputs/$DATE/$SESSION/images"
  mkdir -p "{project-root}/outputs/$DATE/$SESSION/videos"
  mkdir -p "{project-root}/outputs/$DATE/$SESSION/posts"
```

#### 3. Metadata Requirements
Every session MUST include `metadata.json`:

```json
{
  "session_name": "linkedin-carousel-ai-agents",
  "agent": "ai-image-generator",
  "created_date": "2025-10-28",
  "created_time": "01:12:00",
  "platform": "linkedin",
  "content_type": "carousel",
  "topic": "ai agents",
  "files_generated": {
    "images": 3,
    "videos": 0,
    "posts": 1
  },
  "primary_output": "linkedin-carousel-2025-10-28.md"
}
```

```
│   ├── scene1.mp4
│   ├── scene2.mp4
```

## Configuration Variables

### Standard Config Block (All Agents)
```yaml

outputs_session: "{outputs_base}/{current_date}/{session_name}"
```

## Enforcement

This output structure is **MANDATORY** for all BMAD agents as of 2025-10-28.

**Violation Detection:**
- Any output file created outside {project-root}/outputs/ is a violation
- Any session without metadata.json is incomplete
- Any date folder not following MM-DD-YYYY format is non-compliant

**Enforcement Actions:**
- Agents MUST validate output path before writing files
- Agents MUST create date and session folders before generating content
- Agents MUST save metadata.json at session completion
