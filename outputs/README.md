# Outputs Folder Organization

**Last Updated:** 2025-11-05
**Version:** 2.0 (Project-Centric Structure)
**Status:** Active - Mandatory for all agents

---

## Purpose

This folder contains all content generated by the AI-Powered Social Media Agent Team (Jarvis, Zoe, Zoro). Content is organized by **project** (what you created) rather than date (when you created it), with numbered lifecycle stages tracking progress from research to published.

---

## Folder Structure

```
outputs/
├── projects/                        # All active content projects
├── templates/                       # Reusable templates for new projects
├── archive/                         # Projects older than 90 days
└── README.md                        # This file
```

## Project Organization

### Project Naming Convention

**Format:** `{YYYY-MM-DD}-{project-slug}`

**Examples:**
- `2025-11-05-linkedin-post-ai-agents`
- `2025-11-03-youtube-mcp-tutorial-video`
- `2025-11-01-twitter-thread-voice-learning`

**Rules:**
- Date prefix: ISO 8601 format (YYYY-MM-DD) for chronological sorting
- Project slug: lowercase-kebab-case, 1-50 characters
- No spaces, underscores, or special characters
- Descriptive: Indicates platform + content type + topic

## Lifecycle Stages (6 Numbered Folders)

Every project follows the same 6-stage structure:

### **00-session/** - Session Metadata & Logging
**Contains:**
- `metadata.json` - Project metadata, Notion linking, costs, duration, platform-specific file inventory
- `session-log.md` - Timestamped log of all agent actions, handoffs, and platform publishing

**When Created:** Automatically when project starts

---

### **01-research/** - Research Outputs (Shared Across Platforms)
**Contains:**
- `research-brief.md` - Main research findings organized into 5 categories
- `competitive-analysis.md` - Competitor gap analysis (if performed)
- `sources.md` - All source URLs with timestamps and reliability scores

**Populated By:** Jarvis (research-topic, analyze-profile, competitive-analysis workflows)

**Philosophy:** Research is platform-agnostic—same research used for LinkedIn post, Twitter thread, YouTube video, etc.

---

### **02-ideas/** - Content Ideas & Planning (Shared Across Platforms)
**Contains:**
- `idea-cards.md` - Generated idea cards with hooks, outlines, evidence
- `hook-pack.md` - Hook variants (top 5 × 3 variants = 15 hooks)
- `content-calendar.md` - 7-14 day posting schedule (if generated)

**Populated By:** Jarvis (generate-ideas workflow)

**Philosophy:** Ideas are platform-agnostic—same idea adapted for different platforms in 03-drafts/

---

### **03-drafts/** - Platform-Specific Text Content (Iterative Drafts)
**Contains 7 Platform Subfolders:**
- `linkedin/` - LinkedIn posts (post-v1.md, post-v2.md, carousel-post.md)
- `twitter/` - Tweets and threads (thread-v1.md, single-tweet.md)
- `youtube/` - Video scripts (full-video-script.md, short-script.md, description.md)
- `instagram/` - Captions (reel-caption.md, post-caption.md)
- `tiktok/` - TikTok captions (video-caption.md)
- `substack/` - Long-form essays (essay-v1.md, essay-v2.md)
- `facebook/` - Facebook posts (post.md)

**Populated By:** Jarvis (write-post, write-script workflows)

**Naming Convention:** `{content-type}-v{number}.md` for iterations (post-v1.md, post-v2.md)

**Multi-Content Per Platform:** LinkedIn can have post.md + carousel-post.md + video-post.md from same research

---

### **04-media/** - Generated Visual Content (Platform-Agnostic, REUSABLE)
**Contains 2 Simple Subfolders:**
- `images/` - All generated images
  - `thumbnail-main.png` (used for LinkedIn, Twitter, Facebook)
  - `thumbnail-youtube.png` (YouTube-specific CTR variant)
  - `carousel-slide-01.png` through `carousel-slide-10.png`
  - `diagram-architecture.png` (used across platforms)
  - `metadata.json` - Tracks which platforms each image used for
- `videos/` - All generated videos
  - `short-vertical.mp4` (used for YouTube Shorts, Instagram Reels, TikTok)
  - `full-video.mp4` (used for YouTube, LinkedIn)
  - `animated-diagram.mp4` (b-roll, used in multiple videos)
  - `metadata.json` - Tracks which platforms each video used for

**Populated By:** Zoe (create-single-image, create-carousel, create-scene, create-talking-head workflows)

**CRITICAL PHILOSOPHY:** Media is REUSABLE—generate once, reference from multiple platform folders in 05-published/

**Naming Convention:**
- ✅ Descriptive not platform-specific: `thumbnail-main.png`, `short-vertical.mp4`
- ✅ Platform variant suffix only if needed: `thumbnail-youtube.png` (different design from main)
- ❌ Never: `thumbnail-linkedin.png` (implies can't reuse for Twitter)

**Metadata Example:**
```json
{
  "images": [
    {
      "filename": "thumbnail-main.png",
      "quality_score": 9.2,
      "cost": 0.08,
      "used_in_platforms": ["linkedin", "twitter", "facebook"]
    }
  ]
}
```

---

### **05-published/** - Published Content (Merged Final + Published, Per Platform)
**Contains 7 Platform Subfolders:**
- `linkedin/` - LinkedIn published content
  - `post.md` (final version published)
  - `carousel-post.md` (if also published carousel)
  - `url.md` (https://linkedin.com/posts/...)
  - `publish-confirmation.json` (post ID, timestamp)
  - `analytics-2025-11-{06,13,20}.md` (updated over time)
- `twitter/` - Twitter published content
  - `thread.md` (final thread)
  - `urls.md` (all tweet URLs)
  - `publish-confirmation.json`
  - `analytics-2025-11-06.md`
- `youtube/`, `instagram/`, `tiktok/`, `substack/`, `facebook/` - Same pattern

**Populated By:** Zoro (schedule-post primary, publish-*-now backup workflows)

**Philosophy:** "Final approved" = "Published" or "Scheduled to publish"—no intermediate limbo state

**Status Mapping:**
- Notion "Editing" ↔ Local has content in 03-drafts/ and 04-media/
- Notion "Posted" ↔ Local has content in 05-published/ with URLs

**Media References:** 05-published/{platform}/ contains text + references to 04-media/ (no duplication)

---

### **handoffs/** - Agent Coordination Packages
**Contains:**
- `jarvis-to-zoe.json` - Content brief + requirements for visuals
- `zoe-to-zoro.json` - Media file paths + platform specifications for publishing
- `handoff-log.md` - Timeline of all agent handoffs with context

**Purpose:** Audit trail of agent coordination, debugging coordination issues

## Multi-Platform Publishing Example

**Scenario:** Research AI agents → Publish to LinkedIn + Twitter + YouTube Short

**Folder Structure Result:**
```
2025-11-05-ai-agents-multi-platform/
├── 00-session/
│   ├── metadata.json (tracks 3 platforms, $0.53 total cost)
│   └── session-log.md (timeline of all actions)
├── 01-research/
│   └── research-brief.md (SHARED for all 3 platforms)
├── 02-ideas/
│   └── idea-cards.md (10 ideas, SHARED)
├── 03-drafts/
│   ├── linkedin/
│   │   └── post-v1.md (287 words, PAIPS formula)
│   ├── twitter/
│   │   └── thread-v1.md (7 tweets)
│   └── youtube/
│       └── short-script.md (47 second script)
├── 04-media/
│   ├── images/
│   │   ├── thumbnail-main.png (REUSED for LinkedIn + Twitter!)
│   │   └── metadata.json (tracks used_in: ["linkedin", "twitter"])
│   └── videos/
│       ├── short-vertical.mp4 (REUSED for YouTube + could use for Instagram!)
│       └── metadata.json (tracks used_in: ["youtube"])
└── 05-published/
    ├── linkedin/
    │   ├── post.md (references ../../../04-media/images/thumbnail-main.png)
    │   ├── url.md (https://linkedin.com/posts/...)
    │   └── analytics-2025-11-06.md (150 views, 12 likes)
    ├── twitter/
    │   ├── thread.md (references ../../../04-media/images/thumbnail-main.png)
    │   ├── urls.md (7 tweet URLs)
    │   └── analytics-2025-11-06.md (89 views, 8 retweets)
    └── youtube/
        ├── short-script.md (references ../../../04-media/videos/short-vertical.mp4)
        ├── url.md (https://youtube.com/shorts/...)
        └── analytics-2025-11-06.md (340 views, 23 likes)
```

**Key Insight:** ONE thumbnail (`thumbnail-main.png`) served TWO platforms (LinkedIn + Twitter). Media reusability = cost efficiency!

## Finding Content

**Most recent projects:**
```bash
ls -lt outputs/projects/ | head -20
```

**All LinkedIn posts:**
```bash
find outputs/projects/*/03-drafts/linkedin/ -name "*.md"
find outputs/projects/*/05-published/linkedin/ -name "post.md"
```

**All YouTube scripts:**
```bash
find outputs/projects/*/03-drafts/youtube/ -name "*.md"
```

**All generated images:**
```bash
find outputs/projects/*/04-media/images/ -name "*.png"
```

**All published content (any platform):**
```bash
find outputs/projects/*/05-published/*/url.md
```

**Which platforms was image X used for:**
```bash
grep -r "thumbnail-main.png" outputs/projects/*/04-media/images/metadata.json
```
